# 2.1 Preferences: Likes, Dislikes, and Constraints

**Area:** 2. Relationship Memory

Users can track what each person likes, dislikes, avoids, or prefers.

## Capabilities

- Add structured preferences.
- Categorize preferences.
- Mark preferences as confirmed or inferred.
- Add source notes.
- Add confidence level.
- Add dates when preference was learned.
- Search preferences across people.

## Preference Categories

- Food
- Drinks
- Restaurants
- Music
- Movies
- Books
- Colors
- Places
- Travel
- Hobbies
- Clothing
- Brands
- Gifts
- Flowers
- Experiences
- Communication
- Social settings
- Boundaries
- Allergies
- Religious/cultural constraints

## Possible Data Objects

- `Preference`
- `PreferenceCategory`
- `PreferenceSource`
- `PreferenceConfidence`

## Implementation Notes

Preferences should support positive, negative, neutral, and constraint-based records.

Examples:

- likes sushi
- dislikes lilies
- allergic to peanuts
- prefers quiet restaurants
- avoids public attention
